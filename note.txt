File des.py cung cấp các bảng hoán vị, bảng Sbox, ... và các hàm chuyển đổi để file tinh_bias_sbox xử lí
( Kiểu kiểu như dữ liệu đầu vào aka ví dụ để tính toán ấy )

File tinh_bias_sbox nhận 8 sbox từ file des.py, tính toán các xấp xỉ tuyển tính và thu được các bias 
( Output được lưu ở file _output.txt tương ứng, đã tự custom để Sbox 8 ra bias tốt )

Dựa vào output thu được, ta có thể thấy Sbox 8 có bias tốt nhất ~ 28, ta sẽ tấn công vào Sbox này.
File doan_bit_k sử dụng Sbox 8 và tìm ra được xấp xỉ tuyến tính tương ứng, từ đó có thể đoán được phép XOR giữa các bit của khóa K
( Output thu được lưu ở file _output.txt tương ứng, dữ liệu được chạy 10 lần )

File tinh_nhieu_bit_k sẽ sử dụng tất cả các Sbox để tính toán và tìm các xấp xỉ tuyến tính thu được từ Sbox
Dữ liệu thu được cũng dự đoán được một số phép XOR của các bit khóa Khóa
( Output thu được lưu ở file _output.txt tương ứng )

Vấn đề:
- Các bias thu được ở file tinh_nhieu_bit_k quá bé : Có thể do cung cấp chưa đủ số lượng bản rõ - bản mã 
hoặc các thuật toán tính XOR chưa được tối ưu.
- Vẫn dừng ở mức đoán các XOR của các bit K, chưa thể đoán bit K tương ứng.
